# 🌿 대학생 탄소중립 리워드 플랫폼 (걷기 + 리워드 + 인덕이 성장)

## 🧩 1. 개요
대학생의 일상 걷기(캠퍼스 이동)를 탄소중립 실천으로 전환하고, 실천 결과를 포인트/쿠폰 리워드로 즉시 보상하는 플랫폼입니다.
걷기 실천을 게이미피케이션(인하대 마스코트 ‘인덕이’ 성장·커스터마이징)과 연계하여 “계속 쓰게 만드는 재미”를 제공합니다.  


제휴 매장(후문 상권 등)과 연동하여 학생-매장-학교가 함께 이익을 얻는 선순환 구조를 구현합니다.  
운영자를 위한 관리자 대시보드(데이터 모니터링)를 제공합니다.

## 🔍 2. 개발 필요성
인하대 학생 대상 설문조사 결과, 걷기 잠재력(주 4일 이상 학교를 걷는 비율 80%)은 높지만  
탄소중립은 ‘의무감’으로 인식되어 참여 동기가 약함(관심 없음 70%+)이라는 문제를 확인했습니다.  


따라서 “탄소중립 + 의무감”이라는 실패 공식을 “탄소중립 + 확실한 보상 + 지속적인 재미”로 전환하기 위해  
캠퍼스 생활 반경에 최적화된 리워드형 실천 플랫폼이 필요합니다.

## 🎯 3. 세부 목표
- 걷기 기반 리워드(포인트) 시스템 개발: 걸음 데이터 기반 적립, 누적/사용 내역 안정적 관리  
- 쿠폰/제휴 매장 사용 흐름 구현: 쿠폰 발급부터 사용 처리까지 End-to-End 플로우 구축  
- 게이미피케이션 구현: 인덕이 성장·옷장·환경 인터랙션으로 지속 사용 유도  
- 관리자 시스템 구축: 사용자 활동/탄소저감 추정치 등 핵심 지표 모니터링  
- 기능 테스트 및 배포: 시나리오 기반 테스트 후 서버 배포/보안/모니터링 적용  

## 🧠 4. 주요 기능
### 4.1 걷기(만보기) 기반 포인트 적립
**설명**  
사용자는 앱을 켜고 평소처럼 학교를 걸으면 걸음 수가 누적되고, 기준에 따라 포인트가 적립됩니다.  
일/주 단위로 실천을 체감할 수 있도록 누적 현황을 제공합니다.  

**적용 기술**  
- 모바일 센서(걸음 수 수집)  
- 백엔드 포인트 적립 로직(룰 기반)  
- DB 누적 저장 및 트랜잭션 처리  

### 4.2 포인트 사용: 쿠폰 발급 및 사용
**설명**  
사용자는 포인트로 제휴 매장 쿠폰을 발급받아 사용할 수 있습니다.  
쿠폰은 발급 → 사용(검증) → 사용 완료 흐름으로 운영되며, 중복 사용 방지 등 무결성을 보장합니다.  

**적용 기술**  
- 쿠폰 상태 관리(발급/사용/만료 등)  
- 중복 사용 방지(검증 로직)  
- 제휴처 사용 처리 플로우(End-to-End)  

### 4.3 게이미피케이션: 인덕이 성장·옷장(커스터마이징)
**설명**  
포인트는 단순 숫자가 아니라 캐릭터 성장/아이템 구매로 이어져 “지속적인 재미”를 제공합니다.  
예: 과잠 인덕이, 축제 컨셉 등 아이템을 수집·장착하는 옷장 기능  

**적용 기술**  
- 사용자 상태에 따른 이미지 서빙  
- 사용자 커스터마이징(장착/해제)  
- UI 상태 기반 렌더링  

### 4.4 홈 화면 인터랙션(환경 변화 피드백)
**설명**  
앱을 켜면 인하대 캠퍼스 풍경에서 걷기 실천에 따라 하늘이 맑아지고 나무가 자라는 등 시각적 변화를 제공합니다.  
반대로 실천이 부족하면 환경이 오염되는 연출로 행동 변화를 유도합니다.  

**적용 기술**  
- 사용자 활동 데이터 기반 상태 변화 로직  
- 단계별 환경 상태(맑음/성장/오염 등) 렌더링  

### 4.5 관리자 대시보드(운영자)
**설명**  
운영자는 사용자 활동 및 탄소 저감 관련 지표를 한눈에 확인하여 운영 의사결정을 지원받습니다.  

**적용 기술**  
- 관리자 웹(React 등) + 통계/조회 API  
- 모니터링 및 데이터 집계  

### 4.6 데이터 기반 기대효과(탄소 저감 추정치 제공)
**설명**  
“인하대생 2만 명이 주 4일 걷는 행위” 같은 가정에서 연간 탄소 절감량(예: 265톤)을 추정하여  
캠퍼스 차원의 임팩트를 제시합니다.  

## ✨ 5. 기대효과
- 학생: 걷기 습관 형성 + 즉시 보상 + 캐릭터 성장 재미  
- 제휴 매장: 실제 방문/매출 유입  
- 학교/캠퍼스: 탄소중립 실천 확산 + 데이터 기반 성과 제시  
- 운영자: 대시보드 기반 운영 최적화
